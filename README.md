## Концепція та USP

### Назва

**ДТЕК Simulator**

### Жанр

Real-time management/strategy з паузою, елементи survivors-like прогресії

### Платформа

Web (React)

### Високорівневий pitch

> Ти — диспетчер ДТЕК під час найбільшої енергетичної кризи в історії України. Керуй графіками відключень, розподіляй ресурси ППО, відправляй ремонтні бригади та тримай баланс між потребами мільйонів киян — від бабусі з Троєщини до серверів НБУ. Протримайся 5 років наростаючих атак, і, можливо, побачиш перемогу.

### Місія гри

Показати гравцям, що відключення світла — це сувора необхідність, без якої місто може піти в повний блекаут. Гра має просвітницьку мету: дати зрозуміти складність роботи енергетиків під час війни.

### Унікальна торгова пропозиція (USP)

1. **Реальний контекст** — перша гра, що показує війну з перспективи цивільної інфраструктури, а не бойових дій. Історія, яку пережили мільйони українців.

2. **Survivors-like в менеджменті** — знайома механіка "гонки озброєнь" з roguelike/survivors, але замість стрільби — графіки відключень та управління кризами.

3. **Моральні дилеми без правильних відповідей** — кого вимкнути: дитячу лікарню чи насосну станцію? Центр міста перед виборами чи спальні райони взимку?

4. **Чорний гумор + повага** — тон української стійкості та самоіронії.

### Референси

| Гра                                | Що взяти                                             |
| ---------------------------------- | ---------------------------------------------------- |
| This Is The Police                 | Структура дня, дилеми, тон, управління персоналом    |
| 911 Operator / Dispatch            | Реагування на події на мапі в реальному часі         |
| Vampire Survivors / Cookie Clicker | Прогресія складності та апгрейдів                    |
| Frostpunk                          | Атмосфера кризи, важкі рішення, управління ресурсами |

## Цільова аудиторія

### Первинна аудиторія

**Українці 18-45 років**, які пережили/переживають блекаути.

Для них гра буде:

- Способом "відіграти" фрустрацію через розуміння
- Можливістю побачити ситуацію очима енергетиків
- Чорним гумором та катарсисом

### Вторинна аудиторія

**Міжнародні гравці** (потенційно, з локалізацією), яким цікаві:

- Management/strategy ігри
- Ігри з реальним історичним/політичним контекстом
- Унікальний сетинг

### Психографічний профіль

- Люблять системні ігри з жонглюванням ресурсами
- Цінують моральні дилеми без "правильних" відповідей
- Готові до складності та програшів
- Мають 10-30 хвилин на сесію

### Локалізація

- **MVP:** Тільки українська
- **Пізніше:** Можливо англійська

### Потенційні бар'єри

| Бар'єр                          | Рішення                                           |
| ------------------------------- | ------------------------------------------------- |
| "Ненавиджу ДТЕК, не буду грати" | Гра покаже складність їхньої роботи — це і є мета |
| "Занадто болюча тема"           | Баланс гумору та поваги, можливість "перемогти"   |

```
└─────────────────────────────────────────────────────────────┘
```

### Основна задача

**Тримати баланс навантаження через графіки відключень**

```
НАВАНТАЖЕННЯ МЕРЕЖІ:
████████████████████████░░░░░░  78%

▼ Нижче 40% = втрата доходу
▲ Вище 95% = каскадний блекаут
✓ Ціль: 60-85%
```

### Інструменти балансування

1. **Планові графіки** — виставляєш заздалегідь (групи 1.1, 2.3 вимкнути о 18:00)
2. **Аварійні відключення** — реакція на перевантаження (швидко, але злі мешканці)
3. **Перерозподіл** — перекинути навантаження на іншу підстанцію (ризик перегріву)

### Події, що ламають графіки

- Атака пошкодила підстанцію → її групи треба розкидати на інші
- Різке похолодання → +20% споживання за 2 хвилини
- VIP-дзвінок → "увімкніть центр, там захід" → треба когось вимкнути замість
- Лікарня повідомляє про операцію → цю групу не можна вимикати

## Системи гри

### 4.1 Система енергомережі

#### Ієрархія об'єктів

```
КИЇВ (10 реальних районів)
├── Райони (Оболонь, Святошино, Печерськ...)
│   ├── Підстанції (2-4 на район)
│   │   ├── Групи відключення (4-6 на підстанцію)
│   │   │   └── Споживачі (житло, бізнес, критична інфра)
│   │   ├── Трансформатори (температура, знос)
│   │   └── Лінії зв'язку (з іншими підстанціями)
│   └── Джерела (ТЕЦ, підключення до магістралі)
```

#### Райони Києва в грі

1. Оболонський
2. Подільський
3. Шевченківський
4. Святошинський
5. Солом'янський
6. Печерський
7. Голосіївський
8. Деснянський
9. Дніпровський
10. Дарницький

#### Ключові параметри системи

| Параметр                       | Опис                   | Вплив                               |
| ------------------------------ | ---------------------- | ----------------------------------- |
| **Генерація**                  | Скільки МВт доступно   | Ліміт живлення                      |
| **Споживання**                 | Скільки МВт хочуть     | Змінюється від часу, погоди, сезону |
| **Баланс**                     | Генерація - Споживання | Має бути ~0                         |
| **Навантаження лінії**         | % від максимуму        | >100% = перегрів → аварія           |
| **Температура трансформатора** | Градуси                | Перегрів = вихід з ладу             |

#### Механіка перетоків

```
Підстанція А ←──лінія 110кВ──→ Підстанція Б

Якщо А пошкоджена → можна перекинути на Б
АЛЕ: лінія витримує лише X МВт
     Б вже завантажена на Y%
     → Ризик каскадної аварії
```

#### Стани групи відключення

| Стан                 | Іконка | Опис                                         |
| -------------------- | ------ | -------------------------------------------- |
| Увімкнена            | 🟢     | Живиться, споживачі задоволені               |
| Планове відключення  | 🟡     | За графіком, терпіння тікає повільно         |
| Аварійне відключення | 🟠     | Без попередження, терпіння тікає швидко      |
| Пошкоджена           | 🔴     | Фізично неможливо увімкнути, потрібен ремонт |
| Блекаут              | ⚫     | Каскадна відмова, весь район без світла      |

### 4.2 Система загроз та ППО

#### Типи загроз

| Загроза            | Рік появи | Швидкість | Кількість       | Контрзахід         |
| ------------------ | --------- | --------- | --------------- | ------------------ |
| **Шахеди**         | 1         | Повільна  | Масові (10-30)  | Мобільні групи ППО |
| **Крилаті ракети** | 2         | Швидка    | Поодинокі (2-6) | Стаціонарні ЗРК    |
| **Балістика**      | 3         | Миттєва   | Поодинокі (1-3) | Раннє попередження |
| **Кібератаки**     | 3         | -         | Невидима        | IT-захист          |

#### Механіка ППО (автоматична з пріоритетами)

**Перед зміною:**

- Встановлюєш пріоритети захисту (drag & drop):
  1. ТЕЦ-5 (критично)
  2. Підстанція "Північна"
  3. Центр міста
  4. ... решта auto

**Під час атаки:**

- ППО працює автоматично згідно пріоритетів
- Ти бачиш що летить і що перехоплено
- Можеш: терміново евакуювати навантаження з цілі

**Після атаки:**

- Твоя робота: що пошкоджено, як перерозподілити, кого вимкнути, куди слати бригади

#### Наслідки влучань

| Ціль                  | Наслідок                 | Час ремонту  |
| --------------------- | ------------------------ | ------------ |
| Підстанція 110кВ      | Район без світла         | 4-8 годин    |
| ТЕЦ                   | -30% генерації міста     | 2-5 днів     |
| Магістральна лінія    | Ізольовані райони        | 12-24 години |
| Диспетчерська (кібер) | Втрата контролю на 10 хв | -            |

### 4.3 Система персоналу

#### Типи персоналу

| Тип                      | Функція                | Обмеження                       |
| ------------------------ | ---------------------- | ------------------------------- |
| **Ремонтні бригади**     | Відновлюють пошкоджене | Втома, кількість, спеціалізація |
| **Оператори кол-центру** | Обробляють скарги      | Стрес, ефективність падає       |
| **IT-спеціалісти**       | Захист від кібератак   | Потрібні постійно               |
| **Пункти незламності**   | Буфер терпіння району  | Потребують генератори           |

#### Характеристики персоналу

- **Спеціалізація:** лінії / підстанції / трансформатори
- **Досвід:** 1-5 ⭐ (швидкість і якість)
- **Втома:** 0-100% (зростає за зміну, скидається між змінами)
- **Локація:** де зараз знаходяться

#### Механіка бригад

- Відправив бригаду → їде N хвилин → ремонтує M хвилин → повертається
- Можна тримати "на позиції" в гарячій точці
- Втомлена бригада працює повільніше + ризик помилки
- Понаднормові: бригада залишається, але завтра недоступна

### 4.5 Система репутації

#### Групи стейкхолдерів

| Група              | Що хвилює                           | Як впливає                | Критичний поріг               |
| ------------------ | ----------------------------------- | ------------------------- | ----------------------------- |
| **Мешканці**       | Тривалість відключень, попередження | Соцмережі, протести       | <20% = масові протести        |
| **Бізнес**         | Передбачуваність, збитки            | Судові позови, лобіювання | <30% = економічна криза       |
| **Критична інфра** | Будь-яке відключення                | Смерті, аварії, скандали  | <40% = катастрофа             |
| **Влада**          | Імідж, скарги, ЗМІ                  | Бюджет, "прохання"        | <25% = звільнення (game over) |

#### Механіка взаємовпливу

```
Відключив район на 4+ години
    ↓
Мешканці: -15 репутації
    ↓
Скарги у соцмережах
    ↓
ЗМІ підхоплюють
    ↓
Влада: -5 репутації
    ↓
Влада тисне: "увімкніть район депутата"
    ↓
Виконаєш → інші страждають → Мешканці: -10
Відмовиш → Влада: -10, але можливо +5 від мешканців
```

#### Соцмережі як UI-елемент

Фейковий Twitter/Telegram збоку екрану з реакціями:

- Скарги мешканців
- Подяки за попередження
- Новини бізнесу про збитки
- Заяви політиків

## Прогресія кампанії

### Структура: 5 років × 4 сезони = 20 глав

### Рік 1: "Перша зима" (Навчання)

**Наратив:** Ти — новий диспетчер. Масові атаки тільки починаються.

| Сезон | Особливості        | Загрози                |
| ----- | ------------------ | ---------------------- |
| Весна | Туторіал           | Поодинокі шахеди (2-5) |
| Літо  | Кондиціонери       | Шахеди вночі (5-10)    |
| Осінь | Підготовка до зими | Перші крилаті (1-2)    |
| Зима  | КРИЗА: опалення    | Комбіновані атаки      |

### Рік 2: "Адаптація"

**Наратив:** Ворог побачив, що інфраструктура — вразлива ціль.

| Сезон | Особливості     | Загрози                  |
| ----- | --------------- | ------------------------ |
| Весна | Відновлення     | Регулярні шахеди (10-15) |
| Літо  | IT-захист       | Перша кібератака         |
| Осінь | Масована атака  | 30+ шахедів              |
| Зима  | Важча за минулу | Крилаті по ТЕЦ           |

### Рік 3: "Ескалація"

**Наратив:** Ворог вчиться. Атаки стають розумнішими.

### Реалізовані механіки

#### Система підгруп з балансуванням

Кожен район поділений на 3-4 підгрупи. Кожна підгрупа має:

```
┌─────────────────────────────────────────┐
│ ПІДГРУПА                                │
├─────────────────────────────────────────┤
│ • Статус: ON / OFF / DAMAGED            │
│ • Потужність: 25-40 MW                  │
│ • Нагрів: 0-100% (перегорає при 100%)   │
│ • Час вимкнення: секунди                │
│ • Реальні вулиці (для tooltip)          │
└─────────────────────────────────────────┘
```

**Механіка нагріву:**

- Увімкнена група → нагрів +0.4%/сек
- Вимкнена група → нагрів -1.2%/сек
- 100% нагрів = автоматичне пошкодження
- Апгрейд "Охолодження" зменшує швидкість нагріву на 40%

**Механіка скарг:**

- Група вимкнена >45 сек → 😐 незадоволення
- Група вимкнена >90 сек → 😡 скарги, падає репутація
- Настрій мешканців <15% → Game Over (масові протести)
